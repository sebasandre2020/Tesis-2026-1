\section{Conclusiones}

\begin{itemize}
    \item \textbf{Diseño de la arquitectura de red IoT y software:} Se diseñó una arquitectura para la red IoT como para la aplicación de monitoreo, empleando una topología estrella para las conexiones entre los nodos sensores y el gateway. La integración de nodos sensores con comunicación LoRa y una plataforma de visualización basada en tecnologías web modernas y servicios en la nube (Azure) garantiza la escalabilidad, seguridad y facilidad de mantenimiento del sistema.
    
    \item \textbf{Implementación de la red de nodos sensores:} Los nodos sensores se implementaron con éxito utilizando módulos LoRa para la transmisión inalámbrica y sensores integrados de CO\(_2\), temperatura, humedad y material particulado. Este diseño permite capturar y transmitir de forma concurrente múltiples variables ambientales y de confort térmico en tiempo real, validando la robustez de los nodos y el enlace de comunicación en condiciones reales de aula.
    
    \item \textbf{Desarrollo de la plataforma de software y backend:} Se desarrolló una aplicación web funcional (React) para la visualización en tiempo real, respaldada por un backend robusto alojado en Azure Container Apps y una base de datos relacional en Azure SQL Database. El sistema cuenta con alertas que notifican cuando las variables ambientales (CO\(_2\), polvo, temperatura y humedad) superan los límites de salubridad y confort definidos, y la implementación de gráficos históricos facilita la toma de decisiones informadas y la auditoría de la calidad del aire.
    
    \item \textbf{Desarrollo del Gateway IoT Personalizado:} Se logró implementar exitosamente un gateway puente personalizado utilizando un ESP32 y un transceptor LoRa. Este gateway recibe eficazmente las tramas de los nodos sensores y las retransmite hacia el backend en la nube a través de Wi-Fi, superando las limitaciones y complejidades asociadas a los gateways comerciales tradicionales.
    
    \item \textbf{Pruebas de integración exitosas:} Las pruebas del sistema completamente integrado demostraron que el flujo de datos opera de manera correcta y estable, desde la captura en los nodos sensores, pasando por el gateway ESP32, hasta su almacenamiento y visualización en la aplicación web, cumpliendo a cabalidad con los requisitos de diseño estipulados.
\end{itemize}

\section{Recomendaciones}

\begin{itemize}
    \item \textbf{Monitoreo longitudinal y validación estacional:} Se recomienda realizar estudios de monitoreo a largo plazo que abarquen diferentes estaciones del año y condiciones de ocupación variadas. Esto permitirá recopilar un conjunto de datos más robusto para analizar patrones estacionales en la acumulación de CO\(_2\) y ajustar las estrategias de ventilación de manera predictiva.
    
    \item \textbf{Evaluación de despliegues a gran escala:} Con la viabilidad técnica comprobada del gateway personalizado y la arquitectura en la nube, es propicio evaluar la expansión del sistema para cubrir un edificio universitario completo. Se sugiere realizar un estudio de costos y diseñar la topología de red ampliada para soportar múltiples gateways y decenas de nodos distribuidos.
    
    \item \textbf{Optimización de las estrategias de control y ahorro energético:} Aunque ya se implementó el control de ventilación activa priorizando el CO\(_2\) y apoyado en temperatura y humedad con PWM, se recomienda explorar algoritmos de control predictivo basado en modelos (MPC) o control difuso adaptativo en futuros desarrollos para optimizar el consumo de energía en los extractores y minimizar el desgaste mecánico.
    
    \item \textbf{Mejora continua del sistema de alertas:} Evaluar y mejorar el sistema de alertas, incluyendo ajustes dinámicos en los umbrales de CO\(_2\) mediante un perfil de administrador dentro de la aplicación, y diversificar las vías de notificación (SMS, correos electrónicos automatizados, integración con plataformas institucionales) para garantizar una respuesta inmediata por parte del personal de mantenimiento.
\end{itemize}
