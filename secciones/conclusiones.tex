\section{Conclusiones}

\begin{itemize}
    \item \textbf{Diseño de la arquitectura de red IoT y software:} Se demostró la viabilidad de la arquitectura propuesta al integrar nodos sensores, un gateway personalizado y la nube de Azure en una red que no satura el ancho de banda del campus. La topología estrella y el gateway personalizado permitieron enviar la telemetría de forma segura y económica, sin depender de la red de la universidad.

    \item \textbf{Comunicación inalámbrica LoRa:} La comunicación LoRa entre los nodos y el gateway funcionó de manera estable y sin pérdida de datos dentro de las aulas, superando las interferencias y obstáculos físicos como paredes de concreto. Esto confirma que LoRa es adecuado para campus universitarios donde el Wi-Fi tiene problemas de alcance.

    \item \textbf{Plataforma de software e infraestructura en la nube:} La plataforma en la nube basada en Azure SQL y Azure Container Apps procesó correctamente la información de varios salones en paralelo. La integración de WebSockets permitió mostrar alertas de CO\(_2\) en tiempo real en la página web React sin retrasos, y los filtros mensuales optimizaron las consultas a la base de datos para evitar cobros excesivos por el uso de recursos.

    \item \textbf{Estrategias de control comparativas (Sistema de Control):} Se evaluó y comparó el desempeño de las estrategias de control PI/PID, Fuzzy PI (base y agresivo) y LQR para la mitigación del CO\(_2\) en simulaciones y pruebas de campo. Los resultados experimentales demostraron que el controlador LQR ofrece el mejor tiempo de establecimiento, logrando estabilizar la concentración de gas en aproximadamente 50 minutos con alta estabilidad y sin oscilaciones críticas, mientras que la estrategia Fuzzy PI permitió un decremento gradual con menor esfuerzo de ventilación inicial.

    \item \textbf{Pruebas de integración y validación del sistema:} Las pruebas de integración completa validaron satisfactoriamente la operación conjunta y sin fisuras de todo el pipeline de datos, desde la captura física de las cuatro variables ambientales, la modulación LoRa y Wi-Fi en el gateway, el almacenamiento en la nube de Azure, hasta la visualización en la aplicación web React y la activación oportuna de la mitigación activa.

    \item \textbf{Documentación técnica y replicabilidad:} Se estructuró y consolidó una documentación técnica detallada que detalla el diseño electrónico de los nodos, el mapeo de pines adaptadores, el código estructurado en anexos para el firmware local, el gateway y los despliegues IaC, garantizando la reproducibilidad metodológica del sistema en otras instituciones de educación superior.
\end{itemize}

\section{Recomendaciones}

\begin{itemize}
    \item \textbf{Monitoreo longitudinal y validación estacional:} Se recomienda realizar estudios de monitoreo a largo plazo que abarquen diferentes estaciones del año y condiciones de ocupación variadas. Esto permitirá recopilar un conjunto de datos más robusto para analizar patrones estacionales en la acumulación de CO\(_2\) y ajustar las estrategias de ventilación de manera predictiva.

    \item \textbf{Evaluación de despliegues a gran escala:} Con la viabilidad técnica comprobada del gateway personalizado y la arquitectura en la nube, es propicio evaluar la expansión del sistema para cubrir un edificio universitario completo. Se sugiere realizar un estudio de costos y diseñar la topología de red ampliada para soportar múltiples gateways y decenas de nodos distribuidos.

    \item \textbf{Optimización de las estrategias de control y ahorro energético:} Aunque ya se implementó el control de ventilación activa priorizando el CO\(_2\) y apoyado en temperatura y humedad con PWM, se recomienda explorar algoritmos de control predictivo basado en modelos (MPC) o control difuso adaptativo en futuros desarrollos para optimizar el consumo de energía en los extractores y minimizar el desgaste mecánico.

    \item \textbf{Mejora continua del sistema de alertas:} Evaluar y mejorar el sistema de alertas, incluyendo ajustes dinámicos en los umbrales de CO\(_2\) mediante un perfil de administrador dentro de la aplicación, y diversificar las vías de notificación (SMS, correos electrónicos automatizados, integración con plataformas institucionales) para garantizar una respuesta inmediata por parte del personal de mantenimiento.
\end{itemize}
